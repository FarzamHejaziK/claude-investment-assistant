# FAQ

## Setup & basics

**Q: Do I need to know how to code to use this?**
A: No. You need to be comfortable copy-pasting commands into a terminal and editing a markdown file. That's it.

**Q: Is this free?**
A: Yes. Alpaca paper trading is free. Claude Code is free (with usage limits). This template is MIT-licensed. The only thing that costs money is your actual deposits when you switch from paper to live.

**Q: What brokers does this work with?**
A: Alpaca only, out of the box. The daily run uses Alpaca's API to read your portfolio. You could adapt it to other brokers manually (read the proposed orders, place them in Fidelity/Schwab/etc.), but the auto-portfolio reconciliation won't work.

**Q: Do I need a Mac?**
A: No, it works on Linux and Windows too. The Keychain commands in `/investment:setup` are macOS-specific; on other OSes you'll use `secret-tool` (Linux) or Credential Manager (Windows). See `alpaca-setup.md`.

**Q: How much money do I need to start?**
A: Depends on the mode you picked in `/investment:setup`. **Paper:** $0 — paper accounts come with $100K simulated balance automatically. **Live:** Alpaca has no minimum deposit, but you'll want at least $100–$500 so trades are meaningfully sized. Either way, neither paper nor live is a "default" — pick what matches your goal.

## Trading & safety

**Q: Will this place trades automatically?**
A: **No.** This is a deliberate design choice. The tool proposes orders and you execute them yourself in Alpaca. See `safety-and-limits.md` for the full rationale.

**Q: Why doesn't it auto-trade? I'd rather automate it.**
A: Three reasons:
1. **Safety**: a buggy strategy or a bad market day could blow up your account before you notice. Manual execution forces a daily check-in.
2. **Learning**: the point of this tool is to help you build investment discipline. Auto-trading short-circuits that learning loop.
3. **Compliance**: software that auto-trades on a user's behalf has different legal/regulatory implications than software that proposes orders for human review.

**Q: Is this financial advice?**
A: **No.** This is a tool for organizing and tracking your own investment decisions. The rules in your strategy files are *your* rules. Past performance does not guarantee future results. Consult a qualified financial advisor for advice.

**Q: How risky is this?**
A: Depends entirely on your strategy. The dip-buying example is low-risk (broad ETFs, buy-only, mechanical). The active-trading example is much riskier — academic research suggests 70–90% of retail active traders lose to a simple index over multi-year periods.

**Q: Can I lose all my money?**
A: In paper trading: no, it's simulated. In live trading: yes, like any investment, you can lose your principal. Read each strategy's `## Risk rules` section before activating it.

## Strategy questions

**Q: How often should I run `/investment:daily`?**
A: Once per market day, usually in the morning before placing orders. Running it more often is fine but probably noise — most strategies are designed to fire infrequently.

**Q: What if I miss a few days?**
A: No problem. The next run reads the prior journal entries and figures out what was missed. The strategies are designed to be patient — most days nothing fires anyway.

**Q: Can I have multiple strategies running at once?**
A: Yes. Each active strategy in `./strategies/` is processed independently. They share an Alpaca account but each has its own `capital_monthly_usd` budget. Use non-overlapping universes (e.g., different tickers per strategy) to avoid one strategy interfering with another's positions.

**Q: How do I pause a strategy without deleting it?**
A: Open `strategies/<name>.md`, change `status: active` to `status: paused`. Save. The daily run will skip it but keep the file intact.

**Q: How do I delete a strategy entirely?**
A: Either set `status: archived` (keeps history; ignored forever) or just `rm` the file. Either works.

**Q: What if my strategy's rules don't fire for weeks?**
A: That's normal. Dip-buying only fires when markets dip. DCA only fires on the 1st of the month. Active trading only fires when entry signals trigger. Patience is built in. If nothing fires for a year, that's data — the strategy may be too restrictive, and you can re-tune at quarterly review.

**Q: Can I have a strategy that buys options or crypto?**
A: Technically yes (Alpaca supports both), but the example strategies don't, and the default settings (`./claude/settings.json`) deny order-placement tools. You'd need to write your own strategy rules and adjust permissions. Strongly recommended you don't, until you've run cash strategies for a long time.

## File and data questions

**Q: Where does my data live?**
A: All in this repo:
- `strategies/*.md` — your strategy rules (you edit these)
- `journal/<date>.md` — daily memos (auto-generated by `/investment:daily`)
- `journal/<strategy>-research-archive.md` — old research notes (auto-archived)

**Q: Should I commit my journal to git?**
A: Your call. If your repo is private, sure. If public, probably not — the journal contains your portfolio snapshots and trade history. The `.gitignore` has a commented-out section for excluding journal contents; uncomment if you want it local-only.

**Q: How do I back up my data?**
A: It's a git repo. `git push` to GitHub (private repo recommended). Or copy the folder to a cloud drive. Or both.

**Q: Can I edit a journal entry after it's written?**
A: Don't. Journal entries are historical snapshots — they record what was proposed and what was executed on that day. If you find an error, add a correction in the *next* day's entry, don't rewrite history.

**Q: What does the daily memo's "Strategy notes" section mean?**
A: That's where the daily run flags potential improvements to your strategy. Read them, decide if you agree, and if so edit the strategy file yourself. The tool never auto-edits your operational rules.

## Tax & compliance

**Q: Will this prepare my taxes?**
A: No. Alpaca provides a 1099-B at year-end. Use that for taxes. The journal in this repo is a useful supplement (decision history) but not a substitute for the broker's tax reporting.

**Q: Are short-term trades tax-efficient?**
A: No — short-term capital gains (held < 1 year) are taxed at your ordinary income rate, which can be 2–3× the long-term rate. The active-trading example warns about this. If you care about taxes, lean toward buy-and-hold strategies.

**Q: Does this comply with the wash sale rule?**
A: The tool doesn't enforce wash sale rules. If you trade actively and harvest losses, you need to track wash sales manually (or use Alpaca's tax reports). For buy-only strategies, wash sales are usually a non-issue.

## Troubleshooting

**Q: I ran `/investment:daily` but it said "no active strategies." What now?**
A: Either no `.md` files in `./strategies/` have `status: active`, or all of them are `.example.md` (which are ignored). Open one, change status, and re-run.

**Q: The MCP is failing with HTTP 401.**
A: Almost always paper/live mismatch. See `alpaca-setup.md` § Troubleshooting.

**Q: The daily memo is wrong about my positions.**
A: Either the Alpaca data is stale (rare) or you placed a trade outside the proposed list and the strategy ledger drifted. The daily run flags this — read the "Reconciliation note" in the memo. If it persists, run `/investment:daily` again or check Alpaca's dashboard directly.

**Q: Claude Code says my slash command doesn't exist.**
A: Either the file isn't in `.claude/commands/` (check the path), or Claude Code needs a restart. Restart the IDE and try again.

**Q: I want to ask a general question about my strategy. Can I just talk to Claude?**
A: Yes — Claude Code has the full context of this workspace. You don't need a slash command for ad-hoc questions. "Hey, what would happen if VTI dropped 10% next week?" works fine in regular chat.
